# SPEAR: Streaming Positive-nEgative Association Rules with Entropy-Based Regularity

**Authors:** Djouhra Deghbar, Nesrine Dekkiche  
**Institution:** Department of Computer Science, University of Science and Technology Houari Boumediene (USTHB)

This repository contains the full implementation, experiments, paper source, and Weka plugin for the SPEAR algorithm, a proposed contribution to the field of association rule mining in medical data streams.


## What this project is about

Existing algorithms for mining association rules from medical data streams each solve part of the problem but not all of it. Some handle negative associations (two side effects that almost never appear together) but ignore clinical severity. Others use clinical weights but only find positive co-occurrences. Some enforce temporal regularity but use a fragile metric that one outlier can break.

SPEAR addresses all four requirements in a single algorithm:

1. Single-pass streaming on a sliding window
2. Both positive and negative rule mining in one unified output
3. Clinical severity weighting so rare but dangerous side effects are not discarded
4. Entropy-based regularity that measures the full distribution of occurrences instead of relying on a single worst-case gap

The algorithm is evaluated on a real-world dataset of 238,416 drug profiles and compared against three published baselines: Standard Apriori, Jammalamadaka and Budaraju (2025), and MWAR-SW by Ouyang and Huang (2015).


## Dataset

**File:** `medicine_dataset.csv`

**Source:** A publicly available drug side-effect dataset. Each row represents one drug profile and contains up to 42 columns of reported side effects (columns named `sideEffect0` to `sideEffect41`), along with drug metadata.

**Why this dataset was chosen:** It is a real-world medical dataset with a large number of drug profiles (238,416), a wide variety of side effects (over 1,000 unique values), and a natural transaction structure where each drug is a basket of side effects. This makes it directly applicable to association rule mining, and the clinical nature of the data makes the weighting dimension meaningful: some side effects are life-threatening while others are trivial, and the dataset contains both.

**How it is used:** Each drug profile is converted into a transaction (an unordered set of its non-null side effects). These transactions feed all four algorithms under comparison.


## Repository structure

### Root files

**`SPEAR.ipynb`**  
The main Jupyter notebook. It contains the full experimental pipeline:
- Data loading and transaction extraction from the CSV
- Clinical severity weight table and weight assignment
- Shared utility functions (support, weighted support, entropy regularity)
- Implementation of all four algorithms: Apriori, Jammalamadaka, MWAR-SW, and SPEAR
- Quantitative comparison tables and top-rule listings
- Sliding window stability demonstration
- Visualizations (6 plots saved to `spear_comparison.png`)
- Precision@K evaluation saved to `precision_at_k.png`
- Export of all rule sets to the `exported_rules/` folder

Run this notebook from top to bottom with `medicine_dataset.csv` in the same directory.

**`SPEAR_Paper.tex`**  
The LaTeX source of the research paper. It contains the full formal description of the SPEAR algorithm, theoretical definitions, proofs, and experimental results. Compile with `pdflatex` or `xelatex` to produce the paper PDF.

**`AprioriSpear.pdf`**  
The compiled version of the research paper, ready to read without needing a LaTeX installation.

**`medicine_dataset.csv`**  
The dataset used in all experiments. See the Dataset section above.

**`medicine_side_effects.arff`**  
The same dataset converted to ARFF format (Attribute-Relation File Format) for use with the Weka machine learning platform. Generated by the `generate_weka_arff.py` script. Load this file directly into Weka to run the plugin versions of the algorithms.

**`spear_comparison.png`**  
Figure output by Section 15 of the notebook. Contains six comparative plots: items retained per algorithm, rules generated, mean clinical weight of top-10 rules, max-gap vs entropy regularity scatter, runtime, and sliding window stability.

**`precision_at_k.png`**  
Figure output by Section 18 of the notebook. Shows the Precision@K curve for SPEAR and Jammalamadaka, demonstrating that SPEAR surfaces clinically severe rules at the top of its ranking more consistently.


### `exported_rules/`

This folder is generated when Section 19 of the notebook is executed. It contains the complete rule outputs of all algorithms in CSV format for inspection or further analysis.

**`apriori_positive_rules.csv`**  
All positive association rules produced by Standard Apriori, ranked by confidence. Columns include antecedent, consequent, support, confidence, lift, and the annotated clinical weight of the consequent.

**`jammalamadaka_negative_rules.csv`**  
All negative association rules produced by the Jammalamadaka and Budaraju algorithm, ranked by raw negative confidence. Columns include both side effects, their individual supports, joint support, negative confidence, max-gap regularity score, and annotated clinical weights.

**`mwar_sw_positive_rules.csv`**  
All positive weighted association rules produced by MWAR-SW, ranked by weighted confidence. Columns include antecedent, consequent, support, confidence, clinical weight of the consequent, and weighted confidence.

**`spear_rules.csv`**  
The unified rule output of SPEAR, containing both positive and negative rules ranked together by severity score. Columns include antecedent, consequent, rule type (POSITIVE or NEGATIVE), individual supports, joint support, clinical weights of both sides, entropy regularity scores, the rule-specific metric (weighted confidence or weighted negative confidence), and the final severity score.

**`wnar_sw_alerts.csv`**  
Alert output generated by the WNAR-SW Weka plugin. Contains the rules flagged as alerts by the Weka implementation, for comparison with the Python results.


### `weka-plugin/`

A Java plugin that integrates the SPEAR and WNAR-SW algorithms into the Weka machine learning platform. This allows the algorithms to be run inside the Weka GUI or from the Weka command line using the standard ARFF dataset format.

**`src/weka/associations/SPEAR.java`**  
Java source code for the SPEAR algorithm implemented as a Weka association rule miner. Reads an ARFF file, applies weighted support and entropy regularity filtering, and produces unified positive and negative rules.

**`src/weka/associations/WNARSW.java`**  
Java source code for the WNAR-SW (Weighted Negative Association Rules in Sliding Windows) algorithm, also implemented as a Weka association rule miner.

**`SPEAR.jar`**  
Compiled JAR file for the SPEAR Weka plugin. Place this in the Weka `packages` directory or load it via the Weka Package Manager to use the algorithm inside Weka without recompiling.

**`WNARSW.jar`**  
Compiled JAR file for the WNAR-SW Weka plugin.

**`weights.properties`**  
A plain-text configuration file for customizing the clinical severity weights used by the Weka plugins. Each line follows the format `SIDE_EFFECT_NAME=weight` where weight is a value between 0.0 and 1.0. Uncomment and modify entries to override the built-in severity table. Side-effect names are normalized to uppercase with underscores replacing spaces and special characters.

**`build.sh`**  
Shell script to recompile the Java source files and package them into JAR files. Run this if you modify the Java source and need to rebuild the plugins.

**`build/`**  
Output directory for the compiled Java class files, generated when `build.sh` is executed.


## How to run the experiments

**Requirements:** Python 3.8 or later with the following packages installed: `pandas`, `numpy`, `mlxtend`, `matplotlib`.

Install dependencies:

```
pip install pandas numpy mlxtend matplotlib
```

Then open `SPEAR.ipynb` in Jupyter and run all cells from top to bottom. The notebook will:
- Load `medicine_dataset.csv` from the same directory
- Run all four algorithms on a sliding window of 3000 transactions
- Print comparison tables to the output
- Save `spear_comparison.png` and `precision_at_k.png`
- Write all rule CSV files to the `exported_rules/` folder

To use the Weka plugins, load `medicine_side_effects.arff` into Weka and add `SPEAR.jar` or `WNARSW.jar` via the Weka Explorer or Package Manager.
