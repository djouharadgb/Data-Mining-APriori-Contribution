02

Three novel ideas

Not all side effects are equal

Every side effect receives a clinical weight ω ∈ (0, 1] based on severity. Weighted support boosts dangerous items and dampens trivial ones — so "Seizures" at 0.8% frequency survives pruning where standard Apriori would discard it.

wsup(X) = ω(X) × sup(X)
// Seizures at 0.8%: wsup = 1.0 × 0.008 = 0.008 — survives
// Flatulence at 13%: wsup = 0.18 × 0.13 = 0.023 — barely passes

Consistent patterns matter more

Regularity = maximum gap between consecutive appearances in the window. High-weight items must appear steadily to be trusted. The threshold is tightened by weight: maxRegW(x) = maxReg × (1 − ω(x))

What doesn't co-occur, weighted by danger

For retained items A and B, measure how often A appears without B. Two dangerous items avoiding each other scores far higher than two mild ones avoiding each other.

wNegSup(A → ¬B) = ω(A) × ω(B) × (sup(A) − sup(A∪B))
wNegConf = wNegSup / wsup(A)
severity = wNegConf × (ω(A) + ω(B)) / 2

03

How the algorithm runs

A

Process A — Initialization

Read historical records, assign severity weights from clinical table, build inverted index: item → {tid_list, ω, regularity}.

B

Process B — Sliding window (last 3,000 records)

New record arrives → oldest is evicted. Update tid_lists, recompute regularity, prune items failing weighted support or weighted regularity thresholds.

19 of 370 items survive
C

Process C — Alert detection

For every pair (A, B) of retained items: compute joint support via BitSet intersection. If they rarely co-occur, compute wNegSup, wNegConf, severity score. Emit and sort alerts by severity.

The core contribution

No prior work combines weighted support + weighted regularity + negative rules + sliding window. WNAR-SW is the first algorithm to unify all four, enabling clinically-aware, real-time, bidirectional pattern mining on medical event streams.
